# Security Policy

## Supported Versions

Security updates are currently provided for:

- `main` branch (latest)

## Reporting a Vulnerability

Please do not disclose vulnerabilities publicly before triage.

Report via GitHub Security Advisory (preferred) or contact repository owner directly.
Include:

- affected component/file path
- reproduction steps
- impact assessment
- suggested mitigation (if available)

## Response Targets

- Initial acknowledgment: within 3 business days
- Triage decision: within 7 business days
- Remediation timeline: depends on severity and release impact

## Scope Notes

This project interfaces with external stacks (MicroPython, ESP-IDF, esp-matter, connectedhomeip).
If vulnerability root cause is upstream, we will track and mirror mitigation guidance while preparing downstream patches where possible.
